# 要求内容: ローカルディレクトリ読み込み機能

## 変更・追加する機能の説明

ブラウザの File System Access API を使ってローカルのフォルダを選択し、そのディレクトリ構造をインデントテキストに変換してエディタに挿入する機能を実装する。
既存プロジェクトのディレクトリ構造をドキュメント化する用途で使う。

## ユーザーストーリー

- 開発者として、「フォルダを開く」ボタンを押してローカルフォルダを選択したら、そのディレクトリ構造がエディタに反映されてほしい
- 開発者として、読み込み後にエディタを編集してコメントを追記したい
- 開発者として、`node_modules` や `.git` などノイズになるディレクトリは自動的に除外してほしい
- 開発者として、File System Access API 非対応のブラウザではボタンが表示されないようにしてほしい

## 受け入れ条件

- 「フォルダを開く」ボタンを押すとフォルダ選択ダイアログが表示される
- 選択したフォルダのディレクトリ構造がインデントテキストとしてエディタに挿入される
- 以下のディレクトリは自動除外される: `.git`, `node_modules`, `.next`, `dist`, `build`, `__pycache__`, `.DS_Store`
- ディレクトリ・ファイルはアルファベット順、かつディレクトリが先に並ぶ
- File System Access API 非対応ブラウザ（Firefox など）ではボタンが表示されない
- 読み込み中はボタンを非活性にしてユーザーに処理中であることを示す
- ユーザーがダイアログをキャンセルした場合はエディタの内容を変更しない

## 制約事項

- `functional-design.md` に記載の除外リストに準拠する（`.git`, `node_modules`, `.next`, `dist`, `build`, `__pycache__`, `.DS_Store`）
- File System Access API の対応確認は `'showDirectoryPicker' in window` で行う
- 読み込みロジックは `src/lib/localImport.ts` に切り出してコンポーネントから分離する
- 再帰的なディレクトリ読み込みの深さに上限は設けない（パフォーマンス問題が発生した場合に再検討）
