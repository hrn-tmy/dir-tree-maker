# 要求内容: コメント付き出力機能

## 変更・追加する機能の説明

入力テキストの各行に `# コメント` を付けると、プレビューのツリー出力にもそのコメントが並記されるようにする。
これにより、生成したツリーを README や設計書にそのまま貼り付けられる品質にする。

## ユーザーストーリー

- 開発者として、`src/api/  # APIルート` のようにコメントを付けて入力したら、プレビューのツリーにも `# APIルート` が表示されてほしい
- 開発者として、コメントなしの行とコメントありの行が混在していても、それぞれ正しく表示されてほしい
- 開発者として、コピーボタンを押したらコメント付きのツリーテキストをそのまま取得したい

## 受け入れ条件

- `名前  # コメント` 形式の行がパースされ、ツリー記号の右にコメントが表示される
- コメントなしの行は従来通り表示される
- `C#` のようにスペースなしで `#` が続く場合はコメントとみなさない（` #` がコメント区切り）
- コメントはクリップボードコピー時のテキストにも含まれる
- コメントの有無によってツリーの階層構造（記号の並び）が崩れない

## 制約事項

- 既存の `parseTree.ts` と `TreePreview.tsx` を拡張する形で実装する
- `TreeNode` 型に `comment?: string` フィールドを追加する（`functional-design.md` の定義通り）
- Tailwind CSS をこの作業内で導入し、既存の `index.css` による Plain CSS スタイルを置き換える
