# プロダクト要求定義書

## プロダクトビジョンと目的

開発者が設計ドキュメントや README に貼れるディレクトリツリーを、ブラウザ上で素早く・きれいに作れるツールを提供する。
既存のツリー生成サービスとの差別化として「コメント付き出力」「テンプレート」「ローカル読み込み」を備え、設計ドキュメント作成ツールとしての独自ポジションを確立する。

## ターゲットユーザーと課題・ニーズ

**ターゲット:** ソフトウェア開発者（フロントエンド・バックエンド・フルスタック）

**課題:**
- `tree` コマンドの出力をそのまま貼ると説明がなく、読み手にとって意味が伝わりにくい
- 既存の Web ツールはシンプルなツリー生成のみで、コメントや注釈を付けられない
- 新規プロジェクトの構成を設計する際、ゼロから手書きするのが手間

**ニーズ:**
- 各ファイル・ディレクトリに説明コメントを付けてドキュメントに貼りたい
- フレームワークごとの標準構成を雛形として使いたい
- 既存プロジェクトのディレクトリ構造を素早く可視化・ドキュメント化したい

## 主要な機能一覧

1. **リアルタイムツリー編集** — インデントテキストを入力すると即座にツリーが表示される
2. **コメント付き出力** — `# コメント` 記法でファイル・ディレクトリに説明を付けて出力
3. **フレームワークテンプレート** — Next.js / Nuxt.js / SvelteKit / FastAPI / Django など13種の雛形をワンクリックで読み込み
4. **ローカルディレクトリ読み込み** — File System Access API でローカルフォルダを開いてツリー表示
5. **スクロール同期** — 入力エリアとプレビューのスクロール位置をリアルタイムに連動

## 成功の定義

- ページを開いてから30秒以内に、コピー可能なツリーを生成できる
- コメント付き出力・テンプレート・ローカル読み込みの3機能が揃った Web ツールが他に存在しない状態を維持する

## ユーザーストーリー

- 開発者として、インデントでディレクトリ構造を入力したら即座にツリー記号付きでプレビューしたい
- 開発者として、各ファイルにコメントを付けてドキュメントに貼れる形で出力したい
- 開発者として、生成したツリーをワンクリックでクリップボードにコピーしたい
- 開発者として、Next.js などのテンプレートを読み込んで構造をカスタマイズしたい
- 開発者として、ローカルのプロジェクトフォルダを開いてそのままツリーを取得したい
- 開発者として、入力エリアをスクロールしたらプレビューも連動してスクロールしてほしい

## 受け入れ条件

- 1文字入力するたびにプレビューが更新される（明示的な実行操作は不要）
- `名前  # コメント` 形式の入力がツリーのコメントとして出力に並記される
- コピーボタン押下後、プレビューのテキストがクリップボードに入った状態になる
- テンプレート選択後、エディタの内容が選択したテンプレートの雛形テキストに置き換わる
- ローカルフォルダ選択後、そのディレクトリ構造がインデントテキストとしてエディタに入力される
- File System Access API 非対応ブラウザではローカル読み込みボタンが表示されない
- 入力エリアをスクロールするとプレビューが同じ割合でスクロールされ、逆も同様に動作する

## 機能要件

| # | 機能 | 詳細 |
|---|------|------|
| F1 | リアルタイム編集 | 入力ごとにツリーをレンダリング |
| F2 | コメント付き出力 | `名前  # コメント` 形式をパースしてツリーに並記 |
| F3 | クリップボードコピー | プレビューのツリーテキストをワンクリックでコピー |
| F4 | テンプレート選択 | フレームワーク別テンプレートを選択してエディタに挿入 |
| F5 | ローカル読み込み | File System Access API でフォルダを開いてエディタに反映 |
| F6 | スクロール同期 | 入力・プレビューのスクロール位置をパーセンテージで双方向に連動 |

## 非機能要件

- バックエンドなし（静的サイトとして動作）
- モダンブラウザ（Chrome / Edge / Firefox 最新版）で動作
- ローカル読み込みは File System Access API 非対応ブラウザでは非表示にする
