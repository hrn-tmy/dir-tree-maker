# 技術仕様書

## 技術スタック

| 役割 | 技術 | バージョン |
|------|------|-----------|
| UIフレームワーク | React | ^19.2.5 |
| 言語 | TypeScript | ^6.0.3 |
| ビルドツール | Vite | ^8.0.8 |
| スタイリング | Tailwind CSS | ^4.1 ※未導入 |
| 外部ライブラリ | なし（極力使わない方針） | — |

外部ライブラリを追加する場合は、使用目的と理由を明確にして承認を得てから導入すること。

## 開発ツール

| ツール | 用途 | 状態 |
|--------|------|------|
| Vite dev server | ローカル開発・HMR | 導入済み |
| `tsc` | 型チェック | 導入済み |
| ESLint | 静的解析 | 未導入（導入時は Vite デフォルト設定を使用） |
| Prettier | コードフォーマット | 未導入（必要になった時点で導入を検討） |

## デプロイ

- 静的サイトとしてビルド（`npm run build` → `dist/`）
- Vercel または GitHub Pages へのデプロイを想定
- バックエンド・データベース不要

## 技術的制約

- **File System Access API:** Chrome / Edge のみ対応。Firefox・Safari では非対応のため、ローカル読み込みボタンは API 対応ブラウザのみ表示する
- **ブラウザサポート:** モダンブラウザ（Chrome / Edge / Firefox 最新版）を対象とし、IE は対象外
- **外部通信なし:** すべての処理をクライアントサイドで完結させる

## パフォーマンス要件

- テキスト入力からプレビュー更新までを体感上即時に抑える（キーストロークのたびに遅延を感じさせない）
- 入力が大量行（500行程度）になっても UI がブロックしないこと
- debounce などの遅延処理は原則不要とし、必要になった場合のみ導入を検討する
